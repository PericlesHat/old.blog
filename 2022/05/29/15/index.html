前几天面试了华为数通部门的通用软件开发实习生，总结了面试经验，若侵权请联系删除。

  1. 面试官首先介绍了面试流程：

    • 第一部分是自我介绍和项目介绍
    • 第二部分是计算机基础考察
    • 第三部分是笔试复盘 + 手撕一道题

  2. 介绍项目（我用了大概20分钟，面试官问了一些细节和问题）

    面试官：你觉得你的项目的难点是什么？你怎么看待现在很火的人工智能和深度学习？（我的项目之一是深度学习相关的）你提到了 Github，你的模型和 Github 上面那些随手可以找到的有什么区别？

  3. C 和 C++ 的区别？

    我：C++ 是 C 的超集，它相比 C 封装多了非常多的东西，比如 STL 和相关容器，比如保证安全的智能指针和类型转换方法，比如一些集成好的算法像 sort() 。C++ 是面向对象的，有封装继承多态三大特性。（答的比较乱）

    • C++ 是 C 的超集，C 是面向过程的结构化编程语言，C++ 是面向对象的语言。C++ 可以使用引用，C 仅支持指针操作。C++ 面向对象有三种特性，继承封装多态，也对c进行了许多类型安全的封装，比如强制类型转换这些。但是 C++ 的高级特性大都是建立在降低效率的基础上实现的，C 能够以最简便的方式编译，处理低级存储器，因此更适用于偏底层的设计，像嵌入式，单片机这些。

  4. 你提到了智能指针，你能讲讲 shared_ptr 和 unique_ptr 吗？是如何实现的？

    我：shared_ptr 是共享指针，我没有读过源码，但是我知道里面有个计数器会随着指向内存 +1，释放内存 -1。（unique_ptr 答错了，答到了 weak_ptr 的概念上）

    • shared_ptr ：shared_ptr 是一种共享式指针，它引入了一个引用计数的概念，对于一块内存，每增加一个 shared_ptr 指向，其引用计数+1，每有一个 shared_ptr 释放对内存的使用权，即销毁指针或指向其他内存，则引用计数-1，当引用计数为0时，则自动释放其之前指向的内存。
    • unique_ptr： 为了保证同一时间仅能有一个指针来管理一块内存，C++11 引入了unique_ptr，它是一种独占式指针，仅支持一个指针指向该内存。可以使用 std::move 来移动 unique_ptr 指向的内存。

  5. 你了解 std::move 吗？了解模板吗？lambda呢？（还问了几个名词，不懂）

    我：我只知道好像 std::move 是和右值引用相关的…

    面试官：那是用来做什么的呢？

    我：不太清楚

    面试官：这样，你平常有自己学习 C++ 吗？

    我：课堂上的 C++ 没有讲的很深，我自己有读 C++ Primer，还没读完

    面试官：你平时做项目会用 C++ 吗，看你的项目好像是前端和 Python

    我：只有刷题用…

    • std::move 的本质就强制类型转换，它无条件地将实参转为右值引用类型
    • std::move 是将对象的状态或者所有权从一个对象转移到另一个对象，只是转移，没有内存的搬迁或者内存拷贝。

  6. C++ 的一大特性是面向对象，如果让你在 C 中实现面向对象，你会怎么做？

    我：没有了解过… 关于封装，可能是将一个 struct 单独写在一个 .c 文件中，然后让别的文件去引用？

    面试官：差不太多，写在一个文件中可以向其他文件隐藏其中函数的实现细节，仅提供调用的接口。那继承和多态呢？

    我：继承是另外再写一个 .c 文件的结构体，引用基类的结构体吗？然后实现虚函数指针 vptr 并维护虚函数表来实现多态？

    面试官：差不多，无非就是维护一些指向父类函数的指针。

    • 封装：将客观事物抽象为逻辑实体，实体的属性和功能相结合，形成一个有机的整体。并对实体的属性和功能实现进行访问控制，向信任的实体开放，对不信任的实体隐藏。通过开放的外部接口即可访问，无需知道功能如何实现。

      • 可隐藏实体实现的细节。
      • 提高安全性，设定访问控制，只允许具有特定权限的使用者调用。
      • 简化编程，调用方无需知道功能是怎么实现的，即可调用。

      在 .c 文件中封装 struct 结构体，这样可以向其他文件隐藏函数的实现细节，仅仅提供函数调用接口。

    • 继承和多态：继承指形成有层级的类，使得低层级的类可以延用高层级类的特征和方法。多态指一个类的同名方法，在不同情况下的实现细节不同。实现函数指针或虚函数指针，维护虚函数表或指向父类的指针。

  7. 讲讲 STL 容器吧。你知道哪些容器？vector 和 list 的区别是什么？

    我：（如下）

    • vector 底层是数组，会开辟一段连续的内存空间，数组的特点是随机访问时间复杂度为 O(1)，但是增删改的复杂度需要移动大面积的内存，复杂度为 O(n)
    • list 底层为链表，开辟的内存空间不连续，因此访问数据能力为 O(n)，但是增删改操作本身时间复杂度低

  8. 你知道 map 和 unordered_map 的区别吗？什么时候用 map？

    我：map 和 set 底层都是红黑树，这是一种二叉查找树，索引起来效率较慢。unordered_map 和 unordered_set 底层则是哈希，访问时间为 O(1)，效率高

    面试官：那什么时候用 map？

    我：不懂，没用过

    • map：内部实现了一个红黑树，该结构具有自动排序的功能，因此map内部的所有元素都是有序的，红黑树的每一个节点都代表着 map 的一个元素，因此，对于map进行的查找，删除，添加等一系列的操作都相当于是对红黑树进行这样的操作，故红黑树的效率决定了 map 的效率。
    • unordered_map: 内部实现了一个哈希表，因此其元素的排列顺序是杂乱的，无序的
    • map 优点是有序性，很多操作在低时间复杂度下就可以实现，因此效率非常的高，适合有顺序要求的问题，但是占用空间大
    • unordered_map 优点是建立了哈希表，查找速度非常快，但是建表耗费时间

  9. 你提到了哈希，简单介绍下哈希和实现哈希的相关问题吧。

    我：哈希表通过设计散列函数来实现地址映射，查询效率非常高。有可能出现哈希冲突，就是数据经过散列函数得到了相同的 index。解决的方法有二次探测再散列、拉链法等等。

    • 散列表（Hash table，也叫哈希表），是根据键（Key）而直接访问在内存储存位置的数据结构。也就是说，它通过计算出一个键值的函数，将所需查询的数据映射到表中一个位置来让人访问，这加快了查找速度。这个映射函数称做散列函数，存放记录的数组称做散列表。

  10. 你知道 upper_bound 吗，C++ 是如何去找那个边界的？

    我：没听过

    • upper_bound() 函数定义在 <algorithm> 头文件中，用于在指定范围内查找大于目标值的第一个元素，返回的是指向第一个大于给定值的元素的迭代器。从数组的 begin 位置到 end - 1 位置二分查找第一个小于 num 的数字，找到返回该数字的地址，不存在则返回 end。通过返回的地址减去起始地址 begin，得到找到数字在数组中的下标。

  11. 我们来看看你的笔试吧。（让我复盘了我做出来的第一题）

  12. 好，来做道题：leetcode49

    面试官需要共享屏幕并计时 30min。这道题其实不难，但是因为太紧张了，想了几分钟没想到好的办法，我就想着先做出来再说，就暴力了一个比较蠢的哈希遍历方法，时间复杂度 O(nnm)。面试官问我有没有更好的方法，我就说对每个字符串内部排序，然后比较字符串即可，复杂度为 O(nmlogm)，面试官认可了。

  13. 反问：你们的技术栈是什么？

    面试官：C 和 C++，有时候也会用 Lua 来编写 GUI 和 shell。

第二天早上十点钟，梦到自己在踢欧冠决赛的我被 HR 一个电话打起来面试主管面。
